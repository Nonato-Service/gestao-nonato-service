Esta pasta (gestao-tecnica-nonato-service) costuma ser criada por cópia a partir do projeto HTML original.

1. Coloque o projeto original numa pasta com o NOME QUE QUISER (não precisa parecer-se com mais nada).
2. Dentro de gestao-tecnica-nonato-service\biblioteca-tecnica-site:
   - Copie config-origem-copia.example.json para config-origem-copia.json
   - Abra config-origem-copia.json e defina "raizProjetoOriginal" para essa pasta (relativo ao repo gestão ou caminho absoluto).
3. Opcional: execute biblioteca-tecnica-site\instalar-copia.bat para copiar css, js e assets a partir do original.

O projeto na cópia usa IndexedDB separado (ver js\db.js).
